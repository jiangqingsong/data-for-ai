## 二、详细步骤

### 1. 安装 Python 3.9+

Mac 系统自带的 Python 3.7 太老，项目需要 **Python 3.9+**。

```bash
# 通过 Homebrew 安装
brew install python@3.12

# 验证版本
/usr/local/bin/python3.12 --version
# → Python 3.12.13
```

### 2. 克隆代码 + 创建虚拟环境

```bash
git clone git@github.com:jiangqingsong/ExamAI.git
cd data-for-ai/rag-knowledge-qa

# 用 Python 3.12 创建虚拟环境（不要用系统 python3）
/usr/local/bin/python3.12 -m venv .venv
source .venv/bin/activate
```

### 3. 安装 Python 依赖

```bash
pip install --upgrade pip

# chromadb 依赖 numba/llvmlite，在 x86 Mac 上编译容易失败
# 先用预编译包安装 chromadb，再装其他
pip install --only-binary :all: chromadb

# 安装其余依赖
pip install langchain langchain-community langchain-chroma langchain-text-splitters \
  langchain-experimental langchain-huggingface langchain-openai \
  pypdf python-dotenv pandas openai ragas pytest
```

> **已知问题**: `numba` / `llvmlite` 在 x86 Mac Python 3.12 上编译失败。  
> **解决**: 用 `--only-binary :all: chromadb` 安装预编译包，跳过编译步骤。

### 4. 配置 API Key

项目从 `~/.ai-env/llm/config.json` 读取敏感配置（多项目共享，不入库）。

```bash
mkdir -p ~/.ai-env/llm
```

创建 `~/.ai-env/llm/config.json`：

```json
{
  "deepseek": {
    "api_key": "你的火山引擎 API Key",
    "base_url": "https://ark.cn-beijing.volces.com/api/v3",
    "model": "deepseek-v4-pro-260425",
    "temperature": 0.1,
    "max_tokens": 1024
  },
  "embedding": {
    "api_model": "doubao-embedding-text-240715"
  }
}
```

> **跨平台注意**: Windows 上配置文件路径为 `E:/AI-env/llm/config.json`，Mac 上为 `~/.ai-env/llm/config.json`。`config.py` 会自动按优先级查找。

**验证 API 连通性：**

```bash
source .venv/bin/activate
cd backend && python -c "
from openai import OpenAI
from src.config import Config

client = OpenAI(api_key=Config.DEEPSEEK_API_KEY, base_url=Config.DEEPSEEK_BASE_URL)

# 测试 Chat API
r = client.chat.completions.create(
    model=Config.LLM_MODEL,
    messages=[{'role': 'user', 'content': '回复:OK'}],
    max_tokens=5,
)
print(f'Chat API: {r.choices[0].message.content}')

# 测试 Embedding API
r2 = client.embeddings.create(model=Config.EMBEDDING_API_MODEL, input='测试')
print(f'Embedding API: 维度={len(r2.data[0].embedding)}')
"
```

预期输出：
```
Chat API: OK。
Embedding API: 维度=2560
```

### 5. 准备 PDF 数据

将 PDF 教材文件放入 `data/pdfs/` 目录：

```bash
mkdir -p data/pdfs
# 将 PDF 文件拷贝到这个目录
# 示例: 人教版八年级物理上册课本.pdf, 人教版九年级物理全一册.pdf
```

### 6. 重建向量库

```bash
source .venv/bin/activate
cd backend && python -m src.pipeline
```

### 7. 安装前端依赖（如有前端）

```bash
cd frontend && npm install
```

> 需要 Node.js，Mac 上可用 `brew install node` 安装。

### 8. 启动服务

**同时启动后端和前端：**

```bash
# 终端1: 启动后端 (FastAPI, 端口 8000)
source .venv/bin/activate
cd backend && python app_api.py

# 终端2: 启动前端 (Vite, 端口 5173)
cd frontend && npm run dev
```

浏览器打开 **http://localhost:5173/** 即可使用。

### 9. 运行测试验证

```bash
source .venv/bin/activate
cd backend && python -m pytest tests/ -v
# 预期: 26 passed
```

## 三、常见问题

### Q1: 端口 5000 被占用

Mac 上 Control Center / AirPlay Receiver 默认占用端口 5000。

**解决**: 已将后端端口改为 **8000**（`app_api.py` 和 `vite.config.js` 同步修改）。

如果其他端口被占用：
```bash
# 查看端口占用
lsof -i:端口号

# 释放端口
lsof -ti:端口号 | xargs kill -9
```

### Q2: numba/llvmlite 编译失败

x86 Mac 上 Python 3.12 编译 numba 可能失败。

**解决**: `pip install --only-binary :all: chromadb` 使用预编译包。

### Q3: Embedding API 报 "array too long"

火山引擎限制每次最多 256 条文本。

**解决**: `build_vectorstore` 已内置分批处理，每批 100 条。

### Q4: 配置文件找不到

`config.py` 按以下优先级查找：
1. `~/.ai-env/llm/config.json`（Mac 推荐）
2. `E:/AI-env/llm/config.json`（Windows 兼容）

如果配置文件在自定义路径，创建软链接：
```bash
mkdir -p ~/.ai-env/llm
ln -sf /你的自定义路径/config.json ~/.ai-env/llm/config.json
```
